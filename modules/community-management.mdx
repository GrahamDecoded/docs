# Community Management API

The Community Management API enables ERP clients to create and manage professional communities, peer networks, and industry groups. This system integrates with the AAARRR pipeline to drive community-driven awareness, referrals, and business growth.

## Overview

Communities serve as a strategic tool for:
- **Awareness**: Creating organic awareness through peer networks
- **Acquisition**: Community members becoming natural referral sources  
- **Activation**: Regular meetings increasing engagement and product adoption
- **Retention**: Community belonging reducing churn
- **Revenue**: Peer-to-peer sales within communities
- **Referral**: Community members referring other organizations

## Community Types

- **Professional**: Industry-specific professional groups
- **Industry**: Sector-focused communities
- **Peer**: Peer-to-peer networking groups
- **Customer**: Customer success communities
- **Partner**: Partner ecosystem communities

## Core Features

- Community creation and management
- Member management with role-based access
- Meeting scheduling and management
- Resource sharing and collaboration
- Invitation and referral systems
- Analytics and engagement tracking

## Authentication

All endpoints require a valid API token in the Authorization header:

```bash
Authorization: Bearer <your-api-token>
```

## Endpoints

### Create Community

Creates a new community for an ERP client team.

```http
POST /createCommunity
```

**Request Body:**

```json
{
  "teamId": "string",
  "name": "string",
  "type": "professional|industry|peer|customer|partner",
  "description": "string",
  "industry": "string",
  "location": "string",
  "maxMembers": 100,
  "isPrivate": false,
  "membershipFee": 0,
  "currency": "USD",
  "tags": ["legal", "networking"],
  "settings": {
    "allowGuestAccess": false,
    "requireApproval": true,
    "autoArchiveInactive": true,
    "maxMeetingsPerMonth": 4,
    "allowResourceSharing": true,
    "enableChat": false,
    "enableFileSharing": true
  }
}
```

**Response:**

```json
{
  "id": "community-id",
  "message": "Community created successfully",
  "status": "pending_approval"
}
```

### List Communities

Lists communities with optional filtering and pagination.

```http
GET /listCommunities?teamId=<id>&type=<type>&industry=<industry>&status=<status>&limit=20&offset=0
```

**Query Parameters:**

- `teamId` (optional): Filter by ERP client team
- `type` (optional): Filter by community type
- `industry` (optional): Filter by industry
- `status` (optional): Filter by status
- `limit` (optional): Number of results per page (default: 20)
- `offset` (optional): Number of results to skip (default: 0)

**Response:**

```json
{
  "communities": [
    {
      "id": "string",
      "name": "string",
      "type": "professional",
      "description": "string",
      "status": "active",
      "metrics": {
        "totalMembers": 25,
        "engagementScore": 85
      }
    }
  ],
  "pagination": {
    "total": 100,
    "limit": 20,
    "offset": 0,
    "hasMore": true
  }
}
```

## Data Models

### Community

```typescript
interface Community {
  id: string;
  teamId: string;
  name: string;
  type: 'professional' | 'industry' | 'peer' | 'customer' | 'partner';
  description: string;
  industry?: string;
  location?: string;
  maxMembers?: number;
  isPrivate: boolean;
  membershipFee?: number;
  currency?: string;
  status: 'active' | 'inactive' | 'archived' | 'pending_approval';
  tags: string[];
  settings: CommunitySettings;
  metrics: CommunityMetrics;
  createdAt: string;
  updatedAt: string;
  createdBy: string;
}
```

### Community Settings

```typescript
interface CommunitySettings {
  allowGuestAccess: boolean;
  requireApproval: boolean;
  autoArchiveInactive: boolean;
  maxMeetingsPerMonth: number;
  allowResourceSharing: boolean;
  enableChat: boolean;
  enableFileSharing: boolean;
}
```

### Community Metrics

```typescript
interface CommunityMetrics {
  totalMembers: number;
  activeMembers: number;
  totalMeetings: number;
  totalResources: number;
  engagementScore: number;
  referralCount: number;
  referralRevenue: number;
}
```

## Best Practices

### Community Creation
1. **Clear Purpose**: Define specific goals and value proposition
2. **Target Audience**: Identify the right member profile
3. **Engagement Strategy**: Plan regular activities and content
4. **Moderation**: Establish clear guidelines and moderation policies

### Member Management
1. **Onboarding**: Welcome new members with clear expectations
2. **Role Definition**: Clearly define member roles and responsibilities
3. **Engagement Tracking**: Monitor participation and engagement metrics
4. **Recognition**: Acknowledge active contributors and leaders

### Content Strategy
1. **Regular Meetings**: Schedule consistent community gatherings
2. **Resource Sharing**: Encourage knowledge exchange and collaboration
3. **Expert Speakers**: Invite industry experts for presentations
4. **Case Studies**: Share success stories and best practices

## Error Handling

### Common Error Codes

- `400 Bad Request`: Missing required fields or invalid data
- `401 Unauthorized`: Missing or invalid API token
- `403 Forbidden`: Insufficient permissions or access denied
- `404 Not Found`: Community or resource not found
- `500 Internal Server Error`: Server-side error

### Error Response Format

```json
{
  "error": "Error message description",
  "details": "Additional error details if available"
}
```
